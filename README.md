# main-app-journal

本仓库用于归档和发布多个独立 HTML 页面。仓库本身不服务于任何特定域名或网站，仅作为通用页面存储与分发空间。

## 项目简介

`main-app-journal` 是一个轻量级页面集合仓库。所有内容均为静态 HTML 文件，可直接通过 GitHub Pages 或其他静态托管服务访问。适用于存放独立工具页面、信息面板、归档记录等无需后端支持的页面。

## 目录结构说明

```
.
├── index.html          # 主入口页面（可选）
├── pages/              # 存放各个独立 HTML 页面的文件夹
│   ├── page-a.html
│   ├── page-b.html
│   └── ...
├── assets/             # 页面引用的静态资源（CSS、JS、图片等）
│   ├── css/
│   ├── js/
│   └── images/
└── README.md           # 本文件
```

- `pages/`：按需存放不同功能的 HTML 页面，每个页面应保持独立完整。
- `assets/`：存放页面所需的样式、脚本及图片等资源，建议按类型分目录管理。

## 页面归档说明

- 所有页面均为纯前端实现，不依赖后端服务。
- 页面间无强制关联，可单独访问或通过 `index.html` 聚合索引。
- 归档内容不包含任何第三方跟踪、广告或用户数据采集逻辑。
- 如需新增页面，直接在 `pages/` 目录下创建 `.html` 文件，并在 `index.html` 中添加链接（如有需要）。

## 维护说明

- 欢迎提交 Pull Request 添加新的独立页面，但请确保内容不违反 GitHub 社区准则。
- 建议每个页面保持简洁，避免引入过多外部依赖，以提升加载速度与可维护性。
- 若页面引用外部资源（如 CDN 库），请注明来源并确保兼容性。
- 本仓库不承诺对页面内容进行长期更新或兼容性检查，请自行测试。

## 使用方式

1. 克隆本仓库到本地或直接下载 ZIP。
2. 将 `pages/` 下的 HTML 文件打开即可在浏览器中查看效果。
3. 若启用 GitHub Pages，仓库根目录将自动作为网站根目录。
